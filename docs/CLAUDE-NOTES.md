## V2 hard invariants (codified by harness, do not silently break)

- `docs/schemas/provider-evidence.schema.json` status enum stays in
  lockstep with C++ `apply-plan-*` literal emissions in
  `kqoffice/source/ai/provider/ApplyPlanValidator.hxx`. Locked by
  `tests/v2-provider-evidence-schema-test.sh`.
- The defensive `apply-plan-unknown` token must remain present in
  the C++ switch default and absent from the schema enum. Same
  harness asserts both.
- `Provider.cxx` runtime status emissions must be a subset of the
  schema runtime enum (`{ok, provider-error, policy-denied,
  timeout}`). `timeout` is reserved for future backend wiring.
- V1.5 27/27 strict roundtrip baseline must stay green.
  `bin/intelligent-contract-fixtures.sh` is the canonical V1.5
  m3-02 fixture validator and must keep accepting V1.5 fixtures
  without warnings.
- `evidence-record.schema.json` (V1.5 m3-02 capability/diagnostic
  record) and `provider-evidence.schema.json` (V2 per-call audit
  envelope) are **distinct schemas**. Do not collapse them.
- `apply-plan.schema.json` (V1.5 m3-02 capability descriptor:
  per-capability `id` / `undo_group` / `failure_behavior`) and
  `apply-plan-runtime.schema.json` (V2 W3 per-call runtime ApplyPlan
  envelope: `plan_id` / `source_diagnostic_id` / `doc_snapshot_hash`
  / `patches[]`) are **distinct schemas**. Do not collapse them.
  Per-kind patch shape inside `patches[]` lands incrementally with
  each `SwUndoApplyPatch` impl; envelope shape is locked.

## When to ask the user vs. proceed

- Reversible local edits in `docs/`, `tests/`, `bin/`, schema
  fixtures, or workflow YAML — proceed.
- Touching `kqoffice/source/`, `cui/source/dialogs/commandpalette/`,
  `sw/source/uibase/app/docsh*.cxx`, `officecfg/`, `i18npool/` —
  user authorization required. These are gated steps in
  `lane-status.md` (D1, D2, D3 in the open-decisions table).
- `git commit` / `git push` / branch operations — user
  authorization required. Coordinator stops at "ready to commit"
  and reports the explicit-path list.
- Build-tree noise (`workdir/`, `instdir/`, `autom4te.cache/`,
  `config.log`, `config.status`) — never include in any commit
  unless the user explicitly says so.
