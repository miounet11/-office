# 可圈office 功能优先级表

## 1. Must Have
这些功能必须进入当前版本，否则产品无法形成真正可上线的桌面办公 MVP。

| 模块 | 必须具备内容 | 说明 |
|---|---|---|
| 产品形态 | 桌面 Office 优先 | 当前首发方向必须明确，不能同时扩展过多形态 |
| 文档 | 打开 / 编辑 / 保存文档 | 文档是核心高频能力 |
| 表格 | 打开 / 编辑 / 保存表格 | 表格是核心高频能力 |
| 文档基础能力 | 基础排版、插入图片、插入表格 | 满足最常见文档处理需求 |
| 表格基础能力 | 基础单元格格式、常用公式、基础计算 | 满足最常见表格处理需求 |
| 格式兼容 | 支持 `.docx` / `.xlsx` | 首发必须保证基础兼容 |
| 导出 | PDF 导出 | 常见办公交付能力 |
| 界面 | 中文 UI | 降低上手成本 |
| 模板 | 基础模板 | 满足常见场景快速使用 |
| 交付 | 安装包可交付、桌面版本可运行 | 必须能安装、能演示、能试用 |
| 质量 | 基础稳定性、基础易用性 | 首发必须可用，不能只是功能堆砌 |

---

## 2. Should Have
这些功能有价值，但不是当前版本的上线阻塞项。可以在 V1 收尾阶段或紧接着的优化阶段推进。

| 模块 | 建议具备内容 | 说明 |
|---|---|---|
| 模板 | 更完整的常用模板 | 增强实际使用体验 |
| 兼容性 | 更强的中文办公兼容优化 | 提升用户迁移体验 |
| 交互 | 界面进一步简化 | 让产品更容易上手 |
| 引导 | 基础上手说明 / 帮助 | 降低首次使用门槛 |
| 体验 | 常见场景操作优化 | 提升首批用户反馈质量 |

---

## 3. Later
这些内容统一放到后续阶段，当前不进入首发范围，避免拖慢发布。

| 模块 | 后续能力 | 原因 |
|---|---|---|
| 演示 | PPT / 演示能力 | 不属于当前 Docs + Sheets 首发范围 |
| AI | AI 写作、AI 表格、AI 助手 | 明确放在第二阶段 |
| 版式 | OFD | 当前不是首发核心能力 |
| 安全 | 电子签章、国密 | 属于更重的政企能力 |
| 适配 | 深度信创适配 | 当前阶段先不做深度扩展 |
| 协同 | 在线协同编辑 | 会明显增加复杂度 |
| 流程 | 审批流转 | 不属于当前最小可用范围 |
| 生态 | 插件平台 | 属于平台化阶段能力 |
| 企业能力 | 复杂管理逻辑、平台化扩展 | 当前不应拉高系统复杂度 |
| 行业化 | 大量行业定制能力 | 应该在后续按市场需求逐步补充 |

---

## 4. 当前执行规则
当前所有需求判断只看一条：

**如果一个功能能够直接帮助可圈office 更快上线 Docs + Sheets 的桌面 MVP，就进入当前范围；如果它会增加复杂度但不能直接帮助上线，就放到后面。**

一句话执行原则：

**先把必须能上线的做好，再做更强、更深、更复杂的能力。**
